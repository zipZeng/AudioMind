> 版本：V1.0 | 日期：2026-06-16

## 一、模块详细设计

### 1.1 后端 — main.py

```
main.py
│
├── 全局配置
│   ├── app = FastAPI(title="AudioMind")
│   ├── CORS 中间件 (allow_origins=["*"])
│   └── 环境变量读取 (DIFY_API_URL, DIFY_API_KEY, etc.)
│
├── POST /upload
│   │
│   ├── 1. 接收文件
│   │   └── file: UploadFile = File(...), 验证 Content-Type
│   │
│   ├── 2. 调用 Whisper API 转写
│   │   ├── HTTP POST {base_url}/audio/transcriptions
│   │   ├── Header: Authorization: Bearer {OPENAI_API_KEY}
│   │   ├── Body (multipart): file, model="whisper-1", language="zh"
│   │   └── 返回: { text: "转写结果" }
│   │
│   ├── 3. 推入 Dify 知识库
│   │   ├── HTTP POST {DIFY_URL}/datasets/{id}/document/create-by-text
│   │   ├── Header: Authorization: Bearer {DIFY_API_KEY}
│   │   ├── Body (JSON): { name, text, indexing_technique, process_rule }
│   │   └── 返回: { document: { id: "doc_xxx" } }
│   │
│   └── 4. 返回结果
│       └── JSON: { success, text, document_id, message }
│
├── POST /chat
│   │
│   ├── 1. 接收参数
│   │   └── ChatRequest: { query: str, conversation_id: str = "" }
│   │
│   ├── 2. 异步生成器 sse_stream()
│   │   ├── HTTP 流式请求 Dify Chatflow
│   │   │   ├── POST {DIFY_URL}/chat-messages
│   │   │   ├── Body: { query, response_mode: "streaming", user, inputs, conversation_id }
│   │   │   └── 响应: SSE 流 (data: { event, answer, conversation_id, ... })
│   │   │
│   │   └── 逐行 yield 给前端
│   │       └── 保持 data: 格式，每条后跟 \n\n
│   │
│   └── 3. StreamingResponse 返回
│       └── media_type="text/event-stream"
│
├── GET /health
│   └── JSON: { status, dify_configured, whisper_configured }
│
└── GET /
    └── 静态文件 → index.html
```

### 1.2 前端 — index.html

```
HTML 结构:
  .app
  ├── .header        (标题 + 状态灯)
  ├── #messages      (消息列表)
  │   ├── .msg.user   (用户消息气泡)
  │   ├── .msg.ai     (AI 消息气泡)
  │   └── .msg.info   (系统提示)
  ├── #fileBar       (文件预览条)
  ├── .input-area    (输入区域)
  │   ├── #uploadBtn  (上传按钮)
  │   ├── #fileInput  (隐藏的文件 input)
  │   ├── #input      (文本输入框)
  │   └── #sendBtn    (发送按钮)
  └── #fileInput     (隐藏)

JS 状态管理:
  pendingFile: File|null    — 待上传的文件
  streaming: boolean        — 是否正在流式接收答案
  conversationId: string    — Dify 会话 ID（localStorage 持久化）

JS 核心函数:
  sendText()                 — 发送文本 → POST /chat (JSON) → SSE 流处理
  sendAudio()                — 发送音频 → POST /upload → 显示转写结果
  addMsg(role, text)         — 添加消息气泡
  clearFile()                — 清除待上传文件
  newConversation()          — 清空消息 + 重置 conversationId
```

### 1.3 Dify Chatflow

```
节点配置详情:

[开始]                      — 接收 query 字符串
    │ input: query (来自前端用户输入)
    ▼
[知识检索]                   — 从知识库检索相关内容
    │ 知识库: AudioMind 课堂录音
    │ 检索模式: 混合检索（向量 + BM25 关键词）
    │ Top-K: 5
    │ 分数阈值: 0.5
    │ output: 检索到的文本片段列表
    ▼
[LLM]                       — 基于检索结果生成回答
    │ 模型: DeepSeek-V3
    │ Temperature: 0.3
    │ Max Tokens: 1024
    │ System Prompt: (见下方)
    │ Context: 知识检索节点的输出
    │ Memory: 开启（窗口 10 轮）
    ▼
[直接回复]                   — 输出最终答案
    │ output: LLM 的 text → 返回给用户
```

#### System Prompt（完整版）

```
你是一个课堂录音智能助手，帮助大学生复习课程内容。

## 你的能力
- 回答关于课堂内容的任何问题
- 提取作业、考试、重点知识等信息
- 当用户问"上次/昨天/上节课"时，智能匹配最近上传的录音

## 回答规则
1. 始终使用简体中文，语气亲切自然
2. 基于知识库中的转写内容回答，不要编造
3. 如果知识库中没有答案，说"课堂录音中未找到相关信息。
   建议上传更多课堂录音以获得更好的回答。"
4. 回答时注明来源，格式："(来源：XX年XX月XX日课堂录音)"
5. 如果检索到多条相关信息，综合后给出完整答案

## 常见场景
- 问作业：提取作业内容、截止日期
- 问知识点：解释概念，给出课堂讲解要点
- 问考试：提取考试范围、重点、时间
- 问总结：概括某节课/某主题的内容要点
```

## 二、错误处理设计

| 场景 | HTTP 状态码 | 响应 |
|------|------------|------|
| 文件为空 | 400 | `{"error": "未选择文件"}` |
| 文件超 25MB | 413 | `{"error": "文件大小超过 25MB 限制"}` |
| 不支持的格式 | 400 | `{"error": "不支持的音频格式，支持 mp3/wav/m4a/webm/ogg"}` |
| Whisper API 失败 | 502 | `{"error": "语音转写失败: ..."}` |
| Dify API 失败 | 502 | `{"error": "智能问答服务暂不可用: ..."}` |
| Dify API Key 未配置 | 503 | `{"error": "Dify 服务未配置，请联系管理员"}` |
| 请求超时（转写） | 504 | `{"error": "转写超时，请尝试更短的音频"}` |
| query 为空 | 400 | `{"error": "问题不能为空"}` |

## 三、配置项

| 环境变量 | 必填 | 默认值 | 说明 |
|----------|------|--------|------|
| ASR_MODE | 否 | local | ASR 模式：local / api |
| DIFY_API_URL | 是 | https://api.dify.ai/v1 | Dify API 地址 |
| DIFY_API_KEY | 是 | — | Dify Chatflow App API Key |
| DIFY_DATASET_ID | 是 | — | Dify 知识库 ID |
| DIFY_DATASET_KEY | 是 | — | Dify 知识库 API Key（读写） |
| OPENAI_API_KEY | 是 | — | 硅基流动 API Key（ASR） |
| OPENAI_BASE_URL | 否 | https://api.siliconflow.cn/v1 | ASR 服务地址 |
| WHISPER_MODEL | 否 | FunAudioLLM/SenseVoiceSmall | ASR 模型 |
| PORT | 否 | 8080 | 服务端口 |
