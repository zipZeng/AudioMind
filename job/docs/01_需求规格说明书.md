## 一、引言

### 1.1 项目背景

大学生在课堂学习中面临以下痛点：
- 课堂信息量大，笔记难以完整记录
- 课后复习需要反复回听录音，耗时低效
- 想查找"老师上次提到的某个知识点"只能凭记忆

AudioMind 通过语音识别 + RAG 智能检索，将课堂录音转化为可对话的知识库，学生用自然语言提问即可快速定位信息。

### 1.2 项目目标

构建一个基于 Dify 智能体的课堂录音检索系统，实现**录音上传 → 自动转写 → 知识入库 → 智能问答**的完整闭环。

### 1.3 范围

| 在范围内 | 不在范围内 |
|----------|-----------|
| 课堂录音上传与转写 | 实时语音识别（流式） |
| Dify 知识库构建与管理 | 多语言支持（仅中文） |
| 基于 Dify Agent 的智能问答 | 视频转写 |
| Web 前端对话界面 | 移动端 App |
| Docker 云部署 | 用户权限管理（后期迭代） |

## 二、功能性需求

### FR-1 音频上传与转写

| 编号 | 描述 | 优先级 |
|------|------|--------|
| FR-1.1 | 支持上传常见音频格式（mp3, wav, m4a, webm, ogg） | P0 |
| FR-1.2 | 单文件大小限制 25MB（约 30 分钟课堂录音） | P0 |
| FR-1.3 | 调用 Whisper API 将音频转为中文文字 | P0 |
| FR-1.4 | 转写完成后自动推入 Dify 知识库 | P0 |
| FR-1.5 | 前端显示转写进度和结果 | P1 |

### FR-2 知识库管理

| 编号 | 描述 | 优先级 |
|------|------|--------|
| FR-2.1 | 转写文本自动在 Dify 知识库中建立索引 | P0 |
| FR-2.2 | 支持按课程名称标记录音来源 | P1 |
| FR-2.3 | 知识库中可查看已上传的录音转写列表 | P1 |

### FR-3 智能问答

| 编号 | 描述 | 优先级 |
|------|------|--------|
| FR-3.1 | 用户输入自然语言问题，Dify Agent 检索知识库并回答 | P0 |
| FR-3.2 | 回答采用流式输出（字/词级别实时显示） | P0 |
| FR-3.3 | 回答附带信息来源（文件名、时间戳） | P1 |
| FR-3.4 | 无相关知识时明确告知用户 | P0 |
| FR-3.5 | 支持多轮对话上下文记忆（同一会话中） | P2 |

### FR-4 用户界面

| 编号 | 描述 | 优先级 |
|------|------|--------|
| FR-4.1 | 提供 Web 端对话界面 | P0 |
| FR-4.2 | 支持文本输入和文件上传 | P0 |
| FR-4.3 | 适配桌面端和移动端浏览器 | P1 |
| FR-4.4 | 显示知识库中录音文件列表 | P1 |

## 四、用户场景

### 场景 1：课后提问

```
张同学上完软件工程课，用手机录了整节课。
回到宿舍后打开 AudioMind，上传录音文件。
系统转写完成后，他在对话框输入：
  "老师布置了什么作业？"
系统返回：
  "根据今天的课堂录音，老师布置了第三章课后习题 1-5 题，
   要求下周一提交。"
```

### 场景 2：考前复习

```
期末考试前，李同学想回顾"设计模式"相关内容。
他之前上传了多节课的录音。
输入：
  "老师讲过的设计模式有哪些？每个模式用在什么场景？"
系统检索所有课堂录音，返回：
  "在 5 月 10 日的课中讲到单例模式（用于全局配置管理）；
   5 月 17 日讲了工厂模式（用于对象创建解耦）..."
```

### 场景 3：缺席补课

```
王同学因病缺课，同学分享了课堂录音。
他上传录音后：
  "这节课讲了哪些重点？"
  "有没有提到考试范围？"
系统逐一回答，相当于 AI 帮他听了一遍课。
```

## 五、验收标准

| 编号 | 验收项 | 通过条件 |
|------|--------|----------|
| AC-1 | 音频上传转写 | 上传 5 分钟 MP3 录音，3 分钟内输出转写文本 |
| AC-2 | 知识库入库 | 转写文本可被后续问答检索到 |
| AC-3 | 智能问答 | 上传一段含"下周一考试"的录音，提问"什么时候考试？"，系统返回"下周一" |
| AC-4 | 无匹配处理 | 提问无关问题，系统返回"未找到相关信息" |
| AC-5 | Docker 部署 | `docker compose up -d` 后可通过浏览器访问 |
| AC-6 | 文档交付 | 需求规格说明书、系统设计说明书、测试报告、用户手册齐全 |
