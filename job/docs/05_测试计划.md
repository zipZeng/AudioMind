## 二、功能测试用例

### TC-1：音频上传与转写

| 编号 | 用例 | 步骤 | 预期结果 | 负责人 |
|------|------|------|----------|--------|
| TC-1.1 | 上传标准 MP3 | 上传 5 分钟 128kbps MP3 | 返回转写文本，字数 > 0 | 成员4 |
| TC-1.2 | 上传 WAV 格式 | 上传 16kHz mono WAV | 正常转写 | 成员4 |
| TC-1.3 | 上传 M4A 格式 | 上传 iPhone 录音 M4A | 正常转写 | 成员4 |
| TC-1.4 | 文件超过 25MB | 上传 30MB 文件 | 返回 413 错误 | 成员3 |
| TC-1.5 | 上传空文件 | 上传 0 字节文件 | 返回 400 错误 | 成员3 |
| TC-1.6 | 上传非音频文件 | 上传 .txt 文件 | 返回转写失败错误 | 成员3 |
| TC-1.7 | 转写结果入库 | TC-1.1 完成后在 Dify 知识库中搜索 | 能检索到转写内容 | 成员5 |
| TC-1.8 | WebM 格式（浏览器录音） | 浏览器录制 10 秒上传 | 正常转写 | 成员2 |

### TC-2：智能问答

| 编号 | 用例 | 步骤 | 预期结果 | 负责人 |
|------|------|------|----------|--------|
| TC-2.1 | 作业查询 | 上传含"课后把第三章习题做完"的录音后，问"课后作业是什么？" | 返回"第三章习题"相关内容 | 成员5 |
| TC-2.2 | 知识点查询 | 上传含"软件开发生命周期有六个阶段"的录音后，问"软件开发生命周期有哪些阶段？" | 返回六个阶段的具体内容 | 成员5 |
| TC-2.3 | 无匹配 | 问"美国总统是谁？"（知识库无相关内容） | 返回"未找到相关信息" | 成员5 |
| TC-2.4 | 空问题 | 发送空字符串 | 返回 400 错误 | 成员3 |
| TC-2.5 | 流式输出 | 发送任意问题 | 前端逐字显示回答 | 成员2 |
| TC-2.6 | 多条录音检索 | 上传 3 段不同课程的录音，问跨课程问题 | 综合多段录音回答 | 成员5 |
| TC-2.7 | 繁体输入 | 输入"課後作業是什麼？" | 正常检索并回答 | 成员5 |
| TC-2.8 | 超长问题 | 输入 2000 字问题 | 正常处理不崩溃 | 成员3 |

## 三、性能测试

| 编号 | 指标 | 目标值 | 测试方法 |
|------|------|--------|----------|
| TP-1 | 5 分钟音频转写时间 | ≤ 30 秒 | 计时 Whisper API 响应 |
| TP-2 | 问答首字时间 | ≤ 3 秒 | 计时 SSE 第一个 data 事件 |
| TP-3 | 10 并发对话 | 无崩溃、无超时 | ab 或 wrk 压测 /chat |
| TP-4 | 100MB 内存占用 | ≤ 200MB | docker stats |
