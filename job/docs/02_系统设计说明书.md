> 版本：V1.0 | 日期：2026-06-16

## 一、系统架构

### 1.1 总览

```
┌──────────────────────────────────────────────────────────┐
│                     用户浏览器                             │
│               (HTML/CSS/JS 单页应用)                       │
└──────────────────────────┬───────────────────────────────┘
                           │ HTTP (SSE 流式)
                           ▼
┌──────────────────────────────────────────────────────────┐
│                   FastAPI 后端 (:8080)                     │
│                                                          │
│   POST /upload ──── 接收音频 → 调 Whisper → 推 Dify 知识库 │
│   POST /chat   ──── 接收问题 → 转发 Dify Agent → SSE 返回 │
│   GET  /health ──── 健康检查                              │
│   GET  /        ──── 静态文件 (前端 SPA)                  │
└────────┬─────────────────────────────────────┬───────────┘
         │ OpenAI API                          │ Dify API
         ▼                                     ▼
┌──────────────────┐              ┌────────────────────────┐
│  Whisper (ASR)    │              │     Dify 平台            │
│  audio → text     │              │                        │
│                   │              │  📚 知识库              │
│  云端: whisper-1  │              │   - 文档索引            │
│  或本地: faster-  │              │   - Embedding 向量      │
│         whisper   │              │                        │
│                   │              │  🤖 Chatflow Agent     │
│                   │              │   - 知识检索节点         │
│                   │              │   - LLM 推理节点        │
│                   │              │   - 结果整合             │
└──────────────────┘              └────────────────────────┘
```

### 1.2 架构决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 后端框架 | FastAPI | 原生异步支持（SSE 流式），自动生成 API 文档 |
| ASR 方案 | OpenAI Whisper API | 无需 GPU，准确率高，按量付费 |
| 知识库 | Dify 知识库 | 老师要求，内置 Embedding + 分段 |
| 智能体 | Dify Chatflow | 可视化配置，内置知识检索节点 |
| 前端 | 原生 HTML/CSS/JS | 单文件无依赖，快速交付 |
| 部署 | Docker | 一键部署，环境一致 |

## 二、模块设计

### 2.1 后端模块（FastAPI）

```
main.py
├── POST /upload          # 音频上传模块
│   ├── transcribe()      #   调用 Whisper API

```

├── 输入区域
│   ├── 文本输入框
│   ├── 文件上传按钮（隐藏 input）

Chatflow:
  开始 → 知识检索 → LLM → 直接回复

## 三、接口设计

### 3.1 POST /upload

```
请求:
  Content-Type: multipart/form-data
  Body:
    file: <音频文件> (必填, max 25MB)

响应 (200):
  {
    "success": true,
    "text": "转写后的全文...",
    "document_id": "dify_doc_xxx",
    "message": "已转写并存入知识库 (234字)"
  }

响应 (413):
  { "error": "文件大小超过 25MB 限制" }

响应 (500):
  { "error": "转写失败: ..." }
```

### 3.2 POST /chat

```
请求:
  Content-Type: application/x-www-form-urlencoded
  Body:
    query: "昨天布置了什么作业？" (必填)

响应 (200):
  Content-Type: text/event-stream
  data: {"event":"message","answer":"根据课堂录音..."}
  data: {"event":"message","answer":"，老师布置了..."}
  ...
  data: {"event":"message_end"}

响应 (400):
  { "error": "query 不能为空" }
```

### 3.3 GET /health

```
响应 (200):
  {
    "status": "ok",
    "dify_configured": true,
    "whisper_configured": true
  }
```

## 四、数据模型

### 4.1 无自建数据库

AudioMind 不维护自己的数据库。所有数据存储在 Dify 平台：

| 数据 | 存储位置 | 说明 |
|------|----------|------|
| 转写文本 | Dify 知识库 | 自动分段 + Embedding |
| 对话历史 | Dify 会话 | Dify Chatflow 自动管理 |
| 音频原始文件 | 不需要持久化 | 转写后即可丢弃 |

### 4.2 Dify 知识库文档结构

```
document {
  id: "xxx-xxx-xxx",
  name: "2024-03-15_软件工程课.wav",
  text: "转写全文...",
  segments: [
    { id: "seg_1", content: "今天我们来学习软件开发生命周期...", keywords: [...] },
    { id: "seg_2", content: "课后作业是把第三章习题做完...", keywords: [...] },
    ...
  ]
}
```

## 五、关键流程

### 5.1 音频上传流程

```
用户选择文件 → 前端验证大小 → POST /upload
  → FastAPI 接收文件
    → httpx 异步调用 Whisper API
      ← 返回: { text: "转写文本" }
    → httpx 异步调用 Dify 知识库 API
      POST /datasets/{id}/document/create-by-text
      body: { name, text, indexing_technique: "high_quality" }
      ← 返回: { document: { id: "doc_xxx" } }
  ← 返回: { success, text, document_id }
```

### 5.2 智能问答流程

```
用户输入问题 → 前端 POST /chat
  → FastAPI 创建 SSE 流
    → httpx 异步流式调用 Dify Chatflow API
      POST /chat-messages
      body: { query, response_mode: "streaming" }
      ← SSE 流: data: { answer: "..." }
    → 逐个转发给前端
  ← 前端逐字渲染
```
