# AudioMind 项目开发计划

> 版本：V1.0 | 工期：7 天 | 团队：6 人

---

## 一、六人分工

| 成员 | 角色 | 核心职责 | 交付物 |
|------|------|----------|--------|
| **成员1** | 组长 / PM | 项目管理、需求分析、系统设计、PPT制作、答辩 | 需求规格说明书、系统设计说明书、答辩 PPT |
| **成员2** | 前端开发 | Svelte/HTML 前端、对话界面、音频上传页、移动端适配 | 前端页面（index.html）、交互逻辑 |
| **成员3** | 后端开发 | FastAPI 接口（/upload, /chat）、文件处理、异常处理、API 联调 | 后端代码（main.py）、API 文档 |
| **成员4** | 语音识别 | Whisper API 对接、音频格式适配、转写结果处理、转写质量测试 | ASR 模块代码、转写测试报告 |
| **成员5** | RAG 知识库 | Dify 知识库创建、Chatflow 配置、Prompt 调优、检索质量测试 | Dify Chatflow、知识检索测试报告 |
| **成员6** | 运维部署 | Docker 打包、云服务器部署、环境配置、CI/CD（可选） | Dockerfile、docker-compose.yml、部署文档 |

## 二、一周时间表（7天）

| 成员 | 任务 | 产出 |

## 三、风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| Whisper API 不可用（网络/费用） | 中 | 高 | 准备备选方案：Groq Whisper（免费额度） |
| Dify 服务不稳定 | 低 | 高 | 准备本地 Dify 社区版 Docker 部署 |
| 转写准确率不达标 | 中 | 中 | 接受 Whisper 标准准确率（~95%），重点测中文 |
| 团队成员进度落后 | 中 | 中 | 组长每日站会检查，第5天为缓冲期 |
| 云服务器配置不足 | 低 | 低 | 2C4G 足够（计算在云端 API） |
