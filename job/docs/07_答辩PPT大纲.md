# AudioMind 答辩 PPT 大纲

> 建议 8-10 页，15 分钟讲解 + 5 分钟演示

---

## 第1页：封面

```
       AudioMind
       基于 RAG 的课堂录音智能检索系统

       人工智能课程项目

       团队成员：[6人姓名]
       日期：2026年6月
```

---

## 第2页：项目背景（1min）

- **痛点**：课堂信息量大，笔记不全，回听录音费时
- **场景**：上课录音 → 想查"老师布置了什么作业"但不想回听 45 分钟
- **目标**：让 AI 帮学生"听"课，自然语言提问直接得到答案

---

## 第3页：系统演示（5min）⭐核心

**现场操作：**
1. 打开浏览器，访问系统
2. 上传一段测试课堂录音
3. 录音转写完成 → 显示转写文本
4. 提问："老师布置了什么作业？"
5. 系统逐字流式返回答案
6. 再问一个无关问题 → 显示"未找到相关信息"

**准备素材：**
- 一段 2-3 分钟的模拟课堂录音（提前录好）
- 内容包含：知识点讲解 + 作业布置 + 下节课预告

---

## 第4页：技术架构（2min）

```
用户 → Svelte前端 → FastAPI后端 → Whisper(ASR)
                                  → Dify(知识库+Agent)
```

**三个关键点：**
1. Whisper API：语音→文字，准确率 95%
2. Dify 知识库：自动分段 + Embedding 向量化
3. Dify Chatflow Agent：检索 + LLM 推理一体化

强调：**以 Dify 为中心，符合课程 AI 方向要求**

---

## 第5页：核心流程（1min）

```
录音 → 上传 → ASR转写 → 推入Dify知识库
                              ↓
提问 → Dify Agent检索 → LLM推理 → 流式返回答案
```

两个 API 完成全部功能：
- `POST /upload` — 上传+转写+入库
- `POST /chat`   — 提问+检索+回答

## 第7页：关键技术细节（1min）

- **Dify Chatflow 配置**：开始→知识检索→LLM→直接回复
- **System Prompt 设计**：针对"作业查询""知识点""考试范围"三大场景
- **流式 SSE 实现**：后端 httpx 异步转发，前端 ReadableStream 逐字渲染
- **Docker 部署**：一条命令 `docker compose up -d`

## Q&A 准备（常见提问）

| 问题 | 回答要点 |
|------|----------|
| 为什么用 Dify 而不是自建？ | 老师要求；Dify 内置知识库+Agent，开发效率高 |
| 转写准确率如何？ | SenseVoiceSmall 中文准确率约 95%，课堂录音清晰度决定上限 |
| 数据安全性？ | 转写文本存 Dify 云端，原始音频不保存 |
| 和直接用 ChatGPT 的区别？ | 我们的知识库是学生自己的课堂录音，回答基于真实课堂内容 |
| 六个人都写了代码吗？ | 前端、后端、ASR 各写代码；Dify 是配置型工作（Chatflow 拖拽+Prompt）；PM 写文档+PPT；运维写 Dockerfile |
